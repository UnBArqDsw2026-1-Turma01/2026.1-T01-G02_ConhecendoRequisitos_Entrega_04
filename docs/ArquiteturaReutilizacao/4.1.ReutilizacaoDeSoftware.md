## Escolha das reutilizações

A escolha das frentes de reutilização de software no projeto está diretamente ligada ao planejamento que fizemos desde o início do desenvolvimento. Ainda na [Entrega 1](https://github.com/UnBArqDsw2026-1-Turma01/Grupo02_ConhecendoRequisitos_Entrega01), nós já havíamos modelado a identidade visual e desenhado toda a biblioteca de componentes de tela da plataforma dentro do Figma. Como esse design system conceitual já estava pronto, a decisão de construir o frontend em React foi um caminho muito natural, já que o modelo de componentes da biblioteca se alinha perfeitamente à forma como estruturamos as telas no design.

A partir dessa base, conseguimos traduzir a especificação do Figma em componentes de código parametrizados e centralizar as decisões estéticas em Design Tokens globais de CSS. Isso evitou que tivéssemos que reescrever estilos básicos toda vez que criássemos uma nova tela. Além do aproveitamento visual, a equipe também estendeu a reutilização para a camada de persistência: utilizamos a própria modelagem do banco com o Prisma para definir um contrato de dados unificado, garantindo que o backend NestJS e o frontend React compartilhem das mesmas estruturas de dados de forma nativa e sem redundâncias.
